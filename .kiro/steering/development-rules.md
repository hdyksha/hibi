# 開発時の取り決め

## テスト実行について

テストを実行する際は、必ず結果をファイルに出力して確認してください。これにより、長時間のテスト実行やアニメーション出力によるコンテキスト消費を避けることができます。

- `npm run test:save` などのコマンドを使用する
- テスト結果は必ずファイルに保存し、後で確認できるようにする
- テスト実行後は出力されたファイルの内容を確認する
- 長時間実行されるテストのアニメーション出力を避けるため、ファイル出力を優先する

## Git コマンドの使用について

Git コマンドを使用する際は、ユーザー入力を必要としない形で実行してください。

- `git diff` の代わりに `git --no-pager diff` を使用する
- その他のGitコマンドでも、インタラクティブな入力を避けるオプションを使用する
- 例：`git log --no-pager`、`git status --porcelain` など

## 一般的な原則

- コマンド実行時は常に非インタラクティブモードを優先する
- ユーザーの手動入力を必要とする操作は避ける
